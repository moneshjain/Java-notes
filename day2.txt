
2 Tokens in Java

	In programming, a token is the smallest individual unit of source code that is meaningful to a compiler or interpreter. It is a sequence of characters that represents a specific element in the program, such as a keyword, identifier, operator, literal, or punctuation symbol. 
		Every Java program is made up of tokens.

	Java has 6 types of tokens:

		üî∂ 1. Keywords

			These are predefined, reserved words in Java.
			You cannot use them as variable names or method names.

		‚úî Examples
			int, class, static, public, void, if, else, return, try, catch
			‚úî Example in code
			class Student {
				int age;
			}

		üî∂ 2. Identifiers

			Names given by the programmer to:
				Variables // A variable is a memory container used to store data values.
				Methods
				Classes
				Objects
				Interfaces

			‚úî Examples
				age, sum, Student, calculateTotal()
			‚úî Rules
			Cannot start with a number
			Cannot use keywords
			Case-sensitive
			Can contain letters, digits, _, $

		üî∂ 3. Literals

		Constant values written directly in code.
			‚úî Types of literals

			Integer literal ‚Üí 10, 50
			Floating literal ‚Üí 12.5, 3.14f
			Character literal ‚Üí 'A', 'z'
			String literal ‚Üí "Hello"
			Boolean literal ‚Üí true, false
			‚úî Example
			int x = 10;
			String s = "Java";

		üî∂ 4. Operators

			Symbols used to perform operations.
			‚úî Types
			Arithmetic ‚Üí + - * / %
			Relational ‚Üí > < == !=
			Logical ‚Üí && || !
			Assignment ‚Üí = += -=
			Increment / Decrement ‚Üí ++ --
			‚úî Example
			int a = 10 + 5;

		üî∂ 5. Separators / Symbols

			Characters used to separate code elements.
			‚úî Examples
			Semicolon ‚Üí ;
			Comma ‚Üí ,
			Dot ‚Üí .
			Brackets ‚Üí ()
			Curly braces ‚Üí {}
			Square brackets ‚Üí []
			‚úî Example
			System.out.println("Hello");

		üî∂ 6. Comments

			Used to explain code.
			Ignored by the compiler.
			‚úî Types
			Single-line comment
			// This is a comment
			Multi-line comment
			/*
			   Multiple lines comment
			*/

System.out.println() ‚Äî Line-by-Line Explanation
	1Ô∏è‚É£ System

		System is a built-in class in Java.
		It belongs to the java.lang package, so you don‚Äôt need to import it.
		This class contains useful tools like:
			in ‚Üí for input
			out ‚Üí for output
			err ‚Üí for error messages

	2Ô∏è‚É£ System.out

		out is a static variable inside the System class.
		It represents the standard output stream.
		Means: whatever you print using out will go to
		console / terminal / command prompt.

	Technically:

		out is an object of class PrintStream.
		PrintStream has methods like:
		print()
		println()
		printf()

	3Ô∏è‚É£ System.out.println()

		println() is a method of PrintStream.
		It prints the message and moves cursor to the next line.

	üìå Example:
		System.out.println("Hello");
		System.out.println("World");
		Output:
		Hello
		World

	(printed on two different lines)
	
	4Ô∏è‚É£ What println() actually does
		Sends the text to the output stream (out)
		Converts non-string values (int, float, boolean) to string
		Adds a newline at the end (\n)
		
		
		
‚úÖ Difference Between out, in, and err in Java
	1Ô∏è‚É£ System.out ‚Äî Standard Output Stream

		Used to print normal messages to the console.
		Example: results, logs, information, general output.
		Type: PrintStream
		Example:
		System.out.println("Hello, Java!");

	2Ô∏è‚É£ System.in ‚Äî Standard Input Stream

		Used to take input from the keyboard.
		Type: InputStream
		Works with classes like Scanner or BufferedReader.
		Example:
		Scanner sc = new Scanner(System.in);

	3Ô∏è‚É£ System.err ‚Äî Standard Error Stream

		Used to print error messages.
		Also prints to the console, but it's separate from normal output.
		Helps identify errors easily.
		Type: PrintStream (same as out)

about main method	
			
	What is the main method in Java?

		The main method is the starting point (entry point) of every Java program.
		The main method is a public and static method that JVM calls first when the program starts, it returns nothing, and it can take extra input from the command-line.

		Java always begins execution from:
		public static void main(String[] args)

	üîç Line-by-Line Explanation
	
	1Ô∏è‚É£ public
		public means anyone can access this method.
		JVM must be able to access it, so it must be public.
	‚û° Without public, JVM cannot call your main method.

	2Ô∏è‚É£ static

		static means the method belongs to the class, not objects.
		JVM can call it without creating an object of the class.
	‚û° If it was not static, JVM would need to create an object first ‚Äî but JVM doesn‚Äôt do that.

	3Ô∏è‚É£ void

		void means the method does NOT return anything.
		The main method does not give any output back to JVM.

	‚û° It only runs your code.

	4Ô∏è‚É£ main

		main is the name of the method.
		JVM looks for this exact name to start the program.

	‚û° If you change the name, the program won't run.

	5Ô∏è‚É£ (String[] args)

		This is the parameter for the main method.
		String[]
			An array of Strings.
		args
			Variable name (can be anything like a, data, x).
		It stores command-line arguments.

	‚û° If you run your program with extra values, they come inside args.

	Example:
		java MyClass hello 123
	Then:
		args[0] = "hello"
		args[1] = "123"


üü© 2Ô∏è‚É£ Stack Memory ‚Äî Fast Working Table

	Think of the stack as a small table where Java keeps things temporarily.
	
	What is stored in the Stack?
		Local variables (inside methods)
		Method calls (which method is running)
		References (address of objects)
		
üü© 2Ô∏è‚É£ Heap Memory ‚Äî Big Storage Room

	Heap is like a large storage room where Java keeps all objects.

	What is stored in the Heap?
		Objects
		Arrays
		Class objects
		Strings ( some in the String Pool )

	Who manages Heap memory?
	‚û° Garbage Collector (GC)
	
üü® 3Ô∏è‚É£ Method Area / MetaSpace ‚Äî Class Information Storage

	This is the memory where Java stores information about classes.

	What is stored in the Method Area?

		Class names
		Method names
		Method code
		Static variables
		Constant Pool
	‚úî Managed by the JVM
	‚úî After Java 8, this area is called MetaSpace
	‚úî It can grow as needed using OS memory
