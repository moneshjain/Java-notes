
* What is a Programming Language?

		A programming language is a special language used to give instructions to a computer.
		Just like humans understand Hindi or English,
		computers understand only 0s and 1s (binary).
		But writing everything in 0s and 1s is very difficult.
		So we use programming languages (like C, Java, Python, JavaScript) to write instructions in a simple, human-readable way.
		The computer then converts these instructions into machine code (0s & 1s) and executes them.

		Low-Level Languages (Machine Level)
			These languages are closest to the computer hardware.
			They are fast but difficult for humans to understand.
		
		High-Level Languages
			Easy for humans to read and write.
			They are converted into machine code using compiler or interpreter.
			
		Procedural Languages (Structured Programming)
			These languages follow a step-by-step approach.
			Program = set of procedures/functions.
		
		Object-Oriented Programming (OOP Languages)
			Based on objects + classes.
			Useful for large applications.
			
		Functional Programming Languages
			Based on mathematical functions.
			Focus on what to do, not how to do.
			jaise ki java script me hota hai fuctions ka use karte hai 
			
		Markup Languages
			Used to define structure and formatting, not logic.	
			like HTML, XML, JSON
			
		Database Query Languages
			Used for interacting with databases.	
			
			
1. Static Language (Statically Typed Language)

		A static language is a language where type checking happens at compile time.

		‚úî Key Features:

			Variables must have a fixed type
			Type is known before running the program
			Type errors are caught during compilation
			Faster execution (because types are known early)
			More memory-efficient
			Requires explicit type declaration (mostly)

		‚úî Example Languages:
			Java
			C, C++, Rust, Go
			Kotlin, Swift
		‚úî Example in Java:
			int num = 10;    // type fixed as int
			num = "hello";   // ‚ùå Compile-time error
			
2. Dynamic Language (Dynamically Typed Language)

		A dynamic language is a language where type checking happens at runtime.

		‚úî Key Features:
		Variable type is decided at runtime
		Type can change anytime
		Flexible and quick to write
		Errors appear while running the program
		Slower than static languages (because of runtime decisions)

		‚úî Example Languages:
			Python
			JavaScript
			Ruby
			PHP
		‚úî Example in Python:
			x = 10     # x is int
			x = "Hi"   # valid ‚Äî x becomes string
					
			
			
	Static vs Dynamic ‚Äî Comparison Table
	
	Feature			Static Language		Dynamic Language
	Type Checking	Compile-time		Runtime
	Variable Type	Fixed				Can change
	Error Detection	Early				Late
	Speed			Faster				Slower
	Flexibility		Less				More
	Examples		Java, C, C++		Python, JS
	Suitable For	Large systems		Rapid development

What is Java?

	Java is a high-level, object-oriented, platform-independent programming language created by James Gosling at Sun Microsystems (1995).

	Key Features
		Platform Independent ‚Äì Write Once Run Anywhere
		Object-Oriented ‚Äì Everything is based on real-world objects
		Robust & Secure ‚Äì Automatic memory management, bytecode verification
		Multithreaded ‚Äì Can run multiple tasks simultaneously
		Portable ‚Äì Same program works on all OS
		High Performance ‚Äì Uses Just-In-Time compiler
		Distributed ‚Äì Supports RMI, Web services
	Why Java is Still Relevant Today
		Android apps
		Banking software
		Trading applications
		Enterprise Backend systems
		Cloud, Microservices, Spring Boot
		
		
	üîπ 1. Compiler (javac)

		A compiler converts Java source code (.java) into bytecode (.class).
		‚úî What it does:
		Checks syntax
		Reports errors
		Converts human-readable code ‚Üí bytecode
		‚úî Example:
		javac MyProgram.java
		Output: MyProgram.class (bytecode)

	üîπ 2. Interpreter

		JVM has an internal interpreter that executes bytecode line by line.
		‚úî What it does:
		Reads each bytecode instruction
		Executes it one by one
		Makes Java platform independent

	üîπ 3. JIT (Just-In-Time Compiler)

		JIT is part of the JVM.
		It improves performance by converting frequently used bytecode into native machine code.

		‚úî Why JIT?
			Interpretation is slow
			JIT compiles hot code paths ‚Üí fast execution
		‚úî How JIT works:
			JVM interprets code normally
			If a method runs many times, JIT marks it as ‚Äúhot‚Äù
			Converts that method into machine code
			Next time, JVM runs machine code directly ‚Üí faster!

	üîπ 4. JVM (Java Virtual Machine)

		JVM is a virtual computer that runs Java bytecode.

		‚úî What JVM does:
		Loads bytecode
		Verifies bytecode (security)
		Executes bytecode
		Provides memory management (Heap, Stack)
		Performs Garbage Collection
		‚úî JVM = Brain of Java
		Java is platform-independent because bytecode runs on JVM, and JVM is available for every OS (Windows, macOS, Linux).

	üîπ 5. JRE (Java Runtime Environment)

		JRE provides everything needed to run a Java program.

		‚úî JRE contains:

		JVM
		Core libraries (java., javax.)
		Supporting files
		‚úî JRE does NOT contain:
		Compiler
		Development tools
		‚úî Use case:
		If you only want to run a Java program ‚Üí install JRE.
		
	üîπ 6. JDK (Java Development Kit)

		JDK is the complete package for developing Java applications.
		‚úî JDK contains:
		JRE
		JVM
		Libraries
		Compiler (javac)
		Tools (javadoc, jdb, jar)
		‚úî Use case:
		If you want to write + compile + run Java ‚Üí install JDK.


1. Stack Memory (Fast, Temporary Memory)

	Stack memory stores:
		Local variables
		Method parameters
		Function call information
		Primitive data types
		References to objects (not the objects themselves)

	‚úî Key Features:
		Very fast
		Last In First Out (LIFO) structure
		Memory is freed automatically when method ends
		Small size compared to heap
		
		void test() {
			int x = 10;  // stored in stack
			int y = 20;  // stored in stack
		}


2. Heap Memory (Large, Dynamic Memory)

	Heap memory stores:

		Objects
		Arrays
		Instance variables
		All objects created using new
		
	‚úî Key Features:

		Large memory area
		Slower access than stack
		Shared across all threads
		Managed by Garbage Collector

	‚úî Example:
		Car c = new Car();
		
		c ‚Üí stored in stack
		Actual object ‚Üí stored in heap
		
3. Garbage Collector (GC)

	Garbage Collector is a JVM component that automatically removes unused objects from heap memory.
	‚úî When does GC clean objects?
		GC removes an object when:
		‚ùå No reference is pointing to it
		OR
		‚ùå Object becomes unreachable

	‚úî Example:
		Car c = new Car();
		c = null;    // object in heap becomes unreachable
		
	Now GC can delete the object.
